Failure of Success Criterion 3.3.8 and 3.3.9 due to preventing pasting of passwords

ID: F110

Technology: failures

Type: Failure

When to Use

All technologies that require authentication.

Description

Preventing a user from pasting passwords is a failure to meet Success Criteria 3.3.8 and 3.3.9 (unless alternative authentication methods are available). A password in this context could be a password, passcode, passphrase or any string of characters the user has to remember or record to authenticate.

This approach prevents use of common mechanisms to assist a user, such as using a password manager to store and autocomplete the password, or using the device clipboard to copy and paste a password. This forces a user to transcribe the password manually, resulting in a cognitive function test.

Examples

These examples would prevent a user from pasting a password into a password field:

Using onpaste to prevent pasting in a password field

A secure website has an email and password based login form, which prevents pasting using the onpasteevent. This prevents users from manually pasting passwords or using a password manager that is based outside their browser.

      
      <form method="post" action="login">
       <div>
         <label for="email">Email</label>
         <input id="email" type="email" autocomplete="email" ... >
        </div>
        <div>
          <label for="password">Password</label>
          <input id="password" type="password" onpaste="return false" ... >
        </div>
          <input type="submit" value="Login" >
        </div>
      </form>

Using autocomplete to prevent autocompletion in a password field

A secure website has an email and password based login form, which prevents autocompletion using the autocompleteattribute. Although most browsers ignore autocomplete="off" for password inputs, users of older browsers may still be affected.

      
      <form method="post" action="login">
       <div>
         <label for="email">Email</label>
         <input id="email" type="email" autocomplete="off" ... >
        </div>
        <div>
          <label for="password">Password</label>
          <input id="password" type="password" autocomplete="off" ... >
        </div>
          <input type="submit" value="Login" >
        </div>
      </form>

Tests

Procedure

For each form field which accepts password entry:

  1. Check if the structure of the input field(s) prevents the user from pasting or autocompleting their password.
  2. Confirm that no other acceptable authentication methods are present that satisfy Success Criteria 3.3.8 or 3.3.9 (such as an authentication method that does not rely on a cognitive function test).

Expected Results

  • If steps #1 and #2 are true, then this failure condition applies and content fails the Success Criterion.
